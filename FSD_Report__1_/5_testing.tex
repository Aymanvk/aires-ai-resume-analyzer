\chapter{TESTING}
Testing is a crucial part of software development, ensuring that the application runs correctly, is free of bugs, and provides a seamless experience for users. Without proper testing, a system may contain logic errors, broken workflows, or inconsistent data states that frustrate users and undermine reliability. The AIRES system is tested across multiple levels: unit testing, integration testing, system testing, regression testing, and performance testing.  
\noindent
Unit testing checks individual components and functions in isolation---such as the match score algorithm, file type validation, and form input handling---to ensure they produce correct outputs. Integration testing verifies that different modules interact correctly, for example confirming that a resume upload correctly triggers the match score computation across all job listings. System testing evaluates the entire application as a unified system, ensuring all workflows from login to job application function end-to-end. Performance testing ensures the application responds quickly under realistic data loads.
